# feature-planning

## Use when
Scoping a feature before coding, including acceptance criteria, risks, and rollout approach.

## Workflow
1. Restate objective, constraints, and non-goals.
2. Define acceptance criteria and measurable success signals.
3. Propose a file-by-file implementation plan with milestones.
4. Identify risks, edge cases, and migration/rollback strategy.
5. Produce a test plan (unit/integration/e2e) mapped to changed behavior.
6. Wait for explicit approval before implementation.
